# LLM v2.1 评测定义

## 评测对象

评测对象是“岗位原文明示要求的 AI／数字技术强度”的 v2.1 候选模型输出。公司技术程度、行业惯例、年份以及岗位通常可能使用的技术均不属于证据。

## 数据划分

- 开发集 60 条：固定种子抽取，富集 v1–v2 跨阈值变化、同模型重测分歧、规则—模型主阈值冲突、低置信度与 0/1、1/2、2/3 边界。
- 锁定留出集 60 条：从开发集未使用的广告中，按 v2 当前分数和 AI／算法候选分层抽取；不参与提示词调整。
- 两集合的原始广告 ID 无交集。模型不可接收公司、年份、人工抽样原因或另一轮模型输出。

## 单条输出契约（v2.1）

模型必须输出 `canonical_id`、`technology_role`、`strict_ai`、`score`、`boundary_pair`、`evidence`、`reason` 和 `confidence`。代码根据维度重新计算分数；维度与 `score` 不一致、正分无证据、证据不在原文连续出现、ID 缺失/重复或出现额外 ID时，整批拒绝并单条重跑。

最终数据中的 `model_score` 是模型输出在人工覆盖前的冻结副本，不要求模型重复输出一个同义字段。这样既满足逐条保留纯模型分数，也避免让模型生成两个可能自相矛盾的分数字段。

## 指标

- 0–3 精确一致率与相差不超过一级的一致率；
- 主阈值 `score >= 2` 的二分类一致率；
- 二次加权 Cohen’s κ；
- 0、1、2、3 各类 precision、recall、F1 和混淆矩阵；
- 严格 AI（3 分）precision、recall、F1；
- evidence/schema 合法率、请求成功率、token 和估算费用；
- 同模型主编码—盲重测稳定性。

## 防止评测污染

开发集最多进行 3 轮提示词调整，每轮保存提示词指纹、逐条输出和错误类型。冻结提交之后才允许导入留出集人工标签；留出集只评一次，失败不得继续用它调参。合成边界用例来自独立编写的 `config/llm_v2_1_boundary_cases.yaml`，不复制人工评测集原文。

## 实现能力评测

- [ ] 人工开发集可从 XLSX 归一化为 60 条机器可校验参照，原始审核表不被覆盖。
- [ ] 每轮保存完整提示词、指纹、逐条预测、人工对照、混淆矩阵、逐类指标、错误方向和 token 用量。
- [ ] 模型必须先输出 `technology_role` 与 `strict_ai`，其确定映射与 `score` 不一致时整条拒绝。
- [ ] 开发集发布目标为精确一致率 ≥85%、主阈值一致率 ≥95%、二次加权 κ ≥0.85、schema/evidence 合法率 100%。
- [ ] 冻结后 3 次独立开发集运行的主阈值稳定率 ≥95%，且三次均无 schema/evidence 非法输出。
